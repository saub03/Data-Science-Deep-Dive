# Git 명령어 치트시트

## 1) 파일 생성 및 깃초기화
- `mkdir <디렉터리명>` — 야 폴더 새로 파야지 존나 귀찮네
- `touch <파일명>` — 빈 파일 하나 만들어봐, 나중에 채워 ㅠㅠ
- `git init` — 이 폴더를 깃 저장소로 만들어버려, 이제 깃이랑 놀아야지 ㅎㅎ

## 2) 상태 확인
- `git status` — 지금 작업중인 거 뭐 바뀌었나 한눈에 봐라, 개판이면 빨리 add해라
- `git log --oneline --branches --graph` — 커밋 ���스토리 그래프로 보여줘, 브랜치 지랄맞게 꼬인 거 확인 가능
- `git diff` — 아직 add 안 한 변경점 비교해봐, 뭐 수정했는지 까먹었으면 이거 쳐 ㅋㅋ

## 3) 스테이징, 커밋
- `git add <파일명>` — 특정 파일만 스테이징에 올려, 나중에 커밋할 준비해
- `git add .` — 바뀐 거 다 올려버려, 귀찮으면 이게 최고야
- `git commit -m "message"` — 스테이징된 거 커밋해버려, 메시지 적어라 ㅠㅠ
- `git commit -am "message"` — 추적중인 파일만 add+commit 한 방에, 새 파일은 알아서 add해

## 4) 작업 되돌리기
- `git restore <파일명>` — 작업트리 변경사항 롤백해라, 마지막 커밋으로 돌려놔
- `git restore --staged <파일명>` — 스테이징에서 빼버려, 근데 파일은 그대로 두고
- `git reset HEAD^` — 최신 커밋 취소하고 언스테이징, 작업트리는 살림 ㅋㅋ
- `git reset --soft HEAD^` — 커밋만 취소, staged 상태로 놔둬서 수정 더 할 수 있어
- `git reset --hard <커밋해시>` — 그 커밋으로 강제 이동, 뒤에 거 싹 날려버려 (주의: 파일도 위험)
- `git revert <커밋해시>` — 그 커밋 되돌리는 새 커밋 추가, 히스토리 안 건드려서 협업에 안전해

## 5) 브랜치 분기, 병합
- `git branch` — 로컬 브랜치 목록 봐, 뭐뭐 있나
- `git branch <생성할 브랜치명>` — 새 브랜치 만들어, 이동은 따로 해
- `git switch <브랜치명>` — 브랜치로 갈아타, 작업 계속해
- `git merge <브랜치명>` — 현재 브랜치로 다른 브랜치 합쳐, 충돌나면 빡쳐 ㅠㅠ
- `git branch -d <브랜치명>` — 병합 끝난 브랜치 지워버려, 정리하자
- `git cherry-pick <해시>` — 특정 커밋만 골라서 현재 브랜치에 붙여, 유용할 때 많아

## 6) github 연결
- `git remote add origin <URL>` — 깃허브 리포를 origin으로 연결해, 최초 설정
- `git remote -v` — 연결된 리모트 확인해봐, URL 맞나
- `git push -u origin main` — main 브랜치 푸시하고 추적 설정, 처음엔 이거 써
- `git push` — 그냥 푸시해라, upstream 설정돼있으면 알아서 감
- `git pull origin main` — 깃허브에서 최신 꺼 끌어와서 병합, 충돌 조심해
