#  Git & Terminal 필수 명령어 치트시트

---

### 1️⃣ 터미널 기초 (Terminal Basics)
> 파일 시스템을 돌아다니고 관리할 때 사용합니다.

- `pwd`: 현재 내가 있는 폴더의 경로를 출력합니다.
- `ls`: 현재 폴더에 있는 파일 목록을 보여줍니다.
  - `ls -a`: 숨겨진 파일(예: `.git` 폴더)까지 모두 보여줍니다.
- `cd [폴더명]`: 해당 폴더로 들어갑니다.
  - `cd ..`: 한 단계 위(부모 폴더)로 나갑니다.
- `mkdir [폴더명]`: 새로운 폴더를 생성합니다.
- `touch [파일명]`: 비어있는 새로운 파일을 생성합니다.
- `clear`: 터미널 창의 이전 기록을 깨끗하게 지웁니다.

---

### 2️⃣ 버전 관리 기초 (Git Basics)
> 코드의 변경 사항을 기록하고 저장합니다.

- `git init`: 현재 폴더에서 깃 버전 관리를 시작합니다.
- `git status`: 현재 파일들의 상태(수정됨, 스테이지됨 등)를 확인합니다.
- `git add [파일명]`: 파일을 저장 대기 공간(Stage)에 올립니다.
  - `git add .`: 변경된 모든 파일을 한 번에 스테이지에 올립니다.
- `git commit -m "메시지"`: 스테이지에 있는 파일들을 확정 저장합니다.
- `git log`: 지금까지의 저장 기록(커밋 히스토리)을 확인합니다.
  - `git log --oneline`: 로그를 한 줄씩 간결하게 봅니다.

---

### 3️⃣ 브랜치 관리 (Branches)
> 여러 갈래로 나누어 작업할 때 사용합니다.

- `git branch [이름]`: 새로운 브랜치를 만듭니다.
- `git switch [이름]`: 작업할 브랜치를 변경합니다.
- `git merge [이름]`: 다른 브랜치의 작업 내용을 현재 브랜치로 합칩니다.
- `git branch -d [이름]`: 사용이 끝난 브랜치를 삭제합니다.

---

### 4️⃣ 깃허브 협업 (GitHub & Collaboration)
> 원격 저장소와 코드를 주고받습니다.

- `git remote add origin [주소]`: 내 컴퓨터의 저장소와 깃허브 저장소를 연결합니다.
- `git clone [주소]`: 깃허브에 있는 프로젝트를 내 컴퓨터로 그대로 복제해옵니다.
- `git push -u origin main`: 내 커밋들을 깃허브로 업로드합니다.
- `git pull`: 깃허브의 최신 코드를 내 컴퓨터로 가져와 합칩니다.
- `git fetch`: 깃허브의 최신 변경 내역만 확인하고 합치지는 않습니다.

---

### 5️⃣ 되돌리기 및 기타 (Advanced)
- `git restore [파일명]`: 수정한 파일을 마지막 커밋 상태로 되돌립니다.
- `git reset HEAD^`: 가장 최근에 한 커밋을 취소합니다.
- `git revert [커밋해시]`: 특정 커밋의 내용을 취소하는 새로운 커밋을 만듭니다.
